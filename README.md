# 🚀 RAG-from-Scratch: Industrial Grade Retrieval Augmented Generation

A comprehensive, manual implementation of a Retrieval-Augmented Generation (RAG) pipeline designed for educational purposes and architectural depth. **Unlike most implementations, this project avoids high-level abstractions (LangChain, LlamaIndex) to expose the internal mechanics of vector search, reranking, and prompt grounding.**

## 🎯 Project Goal
To build a production-grade "Chat with Documents" system from the ground up, implementing every step of the RAG lifecycle manually to ensure complete control over the retrieval and generation process.

## 🛠️ Technical Architecture

### 🏗️ The Pipeline
The system follows a sophisticated **Advanced RAG** architecture:

**Ingestion Path:**
`PDF` $\rightarrow$ `Text Extraction` $\rightarrow$ `Regex Cleaning` $\rightarrow$ `Sliding Window Chunking` $\rightarrow$ `S-BERT Embedding` $\rightarrow$ `FAISS Index`

**Inference Path:**
`User Query` $\rightarrow$ `Multi-Query Expansion` $\rightarrow$ `Hybrid Search (Dense + Sparse)` $\rightarrow$ `Cross-Encoder Reranking` $\rightarrow$ `Context Injection` $\rightarrow$ `LLM Generation` $\rightarrow$ `Verified Response`

### 📊 Core Engine Components
| Module | Technology | Implementation Detail |
| :--- | :--- | :--- |
| **Vector Store** | FAISS | `IndexFlatIP` with L2 Normalization for Cosine Similarity. |
| **Embeddings** | Sentence-Transformers | `all-MiniLM-L6-v2` (384-dimensional dense vectors). |
| **Retrieval** | Hybrid + RRF | Combines TF-IDF keyword search with Dense vector search using Reciprocal Rank Fusion. |
| **Reranker** | Cross-Encoder | `ms-marco-MiniLM-L-6-v2` for high-precision re-scoring of top candidates. |
| **Generation** | OpenAI / Ollama | Async client with strict grounding prompts to eliminate hallucinations. |
| **Evaluation** | LLM-as-a-Judge | Quantitative metrics: Hit Rate, MRR, and Faithfulness scoring. |

## 🚀 Getting Started

### Prerequisites
- Python 3.10+
- OpenAI API Key (or a local Ollama instance)

### Installation
1. Clone the repository:
   ```bash
   git clone https://github.com/your-username/rag-from-scratch.git
   cd rag-from-scratch
   ```

2. Install dependencies:
   ```bash
   pip install -r requirements.txt
   ```

3. Configure Environment:
   Set your API key in your environment or inside `app.py`:
   ```bash
   export OPENAI_API_KEY='your-api-key-here'
   ```

### Usage
Run the main orchestrator:
```bash
python app.py
```

## 📈 Evaluation Framework
This project implements a rigorous evaluation suite. It uses a "Golden Dataset" to measure:
- **Hit Rate:** % of queries where the correct chunk is retrieved.
- **MRR (Mean Reciprocal Rank):** How highly the correct chunk is ranked.
- **Faithfulness:** A 1-5 score generated by a judge LLM to ensure no hallucinations occur.

## 🧠 Design Decisions (The "Why")
- **Why No LangChain?** To avoid "black-box" abstractions and understand the exact data flow between embeddings and the LLM.
- **Why Hybrid Search?** Dense vectors are great for semantics but fail on specific IDs/Keywords; TF-IDF fills this gap.
- **Why Reranking?** Bi-encoders (FAISS) are fast for retrieval, but Cross-encoders are needed for the final, precise selection.
- **Why Async?** LLM API calls are the primary bottleneck. Asyncio allows the system to handle query expansion and retrieval in parallel.
